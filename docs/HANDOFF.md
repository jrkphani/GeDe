## Patterns (reuse for P7)

- **Derived positions only.** Every node's `{x,y}` = `computeLaneLayout(tier, sort, measured-height)`; node WIDTH never feeds the x-stride. Constrained drag pins x to the lane column + calls the store reorder (sort/rank) — never persists `{x,y}`.
- **Cross-tree shared state → a Zustand store** (separate RF nodes can't share React state): `canvasCompose`, `canvasSatellites`, `canvasCoverage`, `canvasMode`.
- **Stable-id nodes never unmount** → re-home every per-navigation reset to an effect keyed on `canvasSelector`/`canvasIdentity`.
- **Zoom inside a node body:** RF `useStore((s) => s.transform[2] < THRESHOLD)` — a BOOLEAN selector (re-renders only on threshold crossing), never through the reconcile/`measuredSignature` hot path.
- **Beside-the-core geometry keys off the WIDEST MEASURED design-column node** (register/ring/twin share the Design column); feed `node.measured.width` into placement + the re-derive signature. Derive edges from the RECONCILED `nodes`, not the store.
- **A zoom-driven LOD swap of an editable surface must be EDIT-AWARE** — track focus in a **ref** (not state, which re-renders mid-click and cancels click-to-edit). DOM `.contains()` focus checks are unreliable against PORTALLED popovers (Radix) — prefer a stable store flag / `:focus-within` / a width-cap.
- **Pure decision logic belongs in a unit-tested helper, not a flaky e2e** (P6 lesson): `focusPanTarget` extracts the pan DECISION; React Flow keeps the animation-timing race. Reduced-motion + `waitForStableViewport` + geometry `expect.poll` is the e2e de-flake template for anything that MUST stay integration-level.
- **Test focus-race:** after a create, an `onXCreated` rAF×2 focus can steal focus — `await waitForStableViewport(page)` before the next e2e step.
