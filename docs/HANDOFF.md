## What shipped in this deploy (085 + 086)

**085 ‚Äî Design route consolidated editing** (`done`-worthy; supersedes 082 Phase 2). Owner-driven redesign. Three phases, all landed:
- **Phase A ‚Äî canvas physics** (`390198f`): arcs are now **proportional to parameter count**, dots **even-fill** their arc (`slot = span/(m+1)`, no floor), `MAX_DOT_HIT_RADIUS` is **per-layout** (half the true tightest pairwise dot spacing ‚Äî pure even-fill makes within-dimension adjacency the global min). Dots **settle (animate), never jump**. Because context nodes sit at the **centroid of their bound dots**, spreading dots spreads the contexts (fixed the "contexts clustered in a narrow wedge" complaint). An adversarial review caught a real crowded-arc overflow (dots wrapping the seam onto early dots) before commit ‚Äî the fix is the removed floor.
- **Phase B ‚Äî removed the Composer strip** (`254c4e1`): it was ~80% a read-only echo of the register row. Selecting a context now scrolls to + highlights its register row; the justification cell **expands on focus** (roomy prose editing without a separate element). 9 e2e specs that used `.composer-bar` as a compose wait-gate were migrated to the register/canvas markers.
- **Phase C ‚Äî one editing zone** (`a581c74`): rail + register grouped into a single bordered `.editing-zone`; the **canvas moved out to a side visual** (last child, out of the editing tab path). A **rail‚Üíregister tab bridge** (Tab on the rail's last empty phantom ‚Üí the register's new-context phantom) removes the cross-canvas mouse trip. *Scope note: the richer 082 "Tab-on-content-phantom creates + continues into the param phantom" grammar is NOT built ‚Äî within the rail you commit with Enter; that fuller chain is an 082 Phase 1 follow-up.*
- Spec: `docs/issues/085-design-route-consolidated-editing.md`. Decision-aid mockups were built this session (Artifacts).

**086 ‚Äî sync-status miscalibration** (`8ed8d59`): the "Sync error" banner (the real cause of the tester's "buttons don't work" ‚Äî writes/data were actually fine; the banner just screamed) flipped on **any** transient read-path blip (pre-signin `401 missing_token` boot-race, normal long-poll churn) and cleared on any success. Root-cause gap: the ShapeStream's own `subscribe` error channel was never wired to `onError`. Fix: wire it, **hard-ignore** transient/boot-race errors (`isIgnorableReadError`), and **debounce** genuine errors behind a **5s grace** (`SYNC_ERROR_GRACE_MS`) ‚Äî `deriveSyncStatus` stays pure via `errorSince` + `now`. Spec: `086-sync-status-miscalibration.md`.
