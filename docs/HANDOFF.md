## How to work (non-negotiables)

1. **TDD, red first.** Each issue has a *Test-first plan* — write those tests, watch them fail, then implement. Has caught a real bug in nearly every slice.
2. **`npm run verify`** (typecheck → eslint → stylelint → vitest → playwright) must be green before SHIPPED. Husky pre-push runs `verify:fast` (no e2e); GitHub Actions runs full `verify` on push; pre-commit runs `lint-staged`. Currently: 219 unit/component + 19 e2e, typecheck/eslint/stylelint clean.
3. **Schema changes only via `npm run db:generate`** — never hand-write DDL. Rename the generated migration file + update `meta/_journal.json`'s `tag` to match project convention (`0000_init`, `0001_dimensions`, …) — the runner globs by filename, not the journal.
4. **Layer boundaries are lint-enforced**: components can't import `src/db/**`; writes flow component → store → `src/db/mutations.ts`. Also enforced: no raw `<button>`/`<input>`/`<select>` or direct `@radix-ui`/`cmdk` outside `src/components/ui/`; no hardcoded CSS colors (stylelint).
5. **Ship ritual**: Status → SHIPPED, `git mv` to `docs/issues/done/`, update README index row (✅), commit with issue number, push. One commit per issue.
6. **References lines in issues are binding** — deviating from a cited SPEC/STYLE_GUIDE/SITEMAP section is a spec discussion, not an implementation choice.
7. **New UI composes `src/components/ui/`** primitives — don't hand-roll. `EditableGrid` is the one grid primitive exempt (owns its own cells). See ADR-0007.

## Architecture facts you'll need

- **DB**: PGlite, `idb://gede` in browser, `memory://` in tests. `getDatabase()` (`src/db/client.ts`) is a **singleton** — never instantiate two PGlites on the same idb dir (StrictMode double-mount deadlocks it).
- **Stores** (Zustand): `projects`/`dimensions`/`parameters`/`contexts`/`status` share one db handle via `src/store/database.ts`. Reset helpers exist per store for tests.
- **Shell slots**: tiers fill the context bar via `<ContextBar>…</ContextBar>` (portal; collapses when empty). All feedback goes through `useStatusStore.announce(message, {label, run})` — no toasts.
- **Routing**: `src/shell/routes.ts` (pure parse/serialize) + `src/shell/router.ts`. Add new route shapes to the type + both functions + the round-trip test.
- **Undo/redo**: `useCommandLogStore` — every store action pushes `{label, undo, redo}`; `batch(label, fn)` groups a multi-call gesture into one step. ⌘Z/⇧⌘Z in `AppShell`, capture-phase. Session-scoped, cleared on project switch. Don't add per-feature undo buttons.
- **UI primitive layer** (`src/components/ui/`, ADR-0007): Tailwind v4, but `tokens.css` stays the source of truth via `theme-bridge.css` (one-directional). Primitives: `Button`, `InlineEdit`/`PhantomInput`, `MultilineEdit` (auto-grow, commits empty), `Popover`/`Command`/`Swatch`/`Input`.
- **EditableGrid** (`src/components/EditableGrid.tsx`, ADR-0004): shared grid for all tiers. Column config `{ id, header, cell: {kind, ...} }`; callers own `getValue`/`onCommit`, grid owns editing/navigation. `multiline` kind auto-grows, clamps to 2 lines display. Every `<tr>` carries `data-row-id`; `onRowClick?` fires alongside a cell's own click. **Never inline the `cell` function per-render** — see Gotchas. Tier 013/014 should reuse unchanged.
- **ContextRegister**: dynamic columns from live dimensions. Draft/complete computed live via `isComplete()`, never stored.
- **Canvas** (`src/components/Canvas.tsx` + `src/domain/canvasLayout.ts`): `layout(input) → geometry` is pure (zero React/store imports). Fully presentational — `selectedContextId`/`onSelect` are just props; `DesignSurface` owns the store call + announcement, so `Canvas`'s tests never need a real store/DB.
- **Composer** (`src/components/Composer.tsx`, read mode only until 010): presentational like `Canvas`. No composer slot in the shell chrome — it's main-surface content below `.design-surface-row`.

## Gotchas already paid for (don't rediscover)

- **RTL auto-cleanup doesn't run** without vitest globals — `src/test/setup.ts` registers it manually. Component tests need `// @vitest-environment jsdom` as line 1.
- **Radix popovers close on Escape at document level** — in-place editors inside popovers need an `onEscapeKeyDown` guard (see DimensionManager) to honor the Esc order.
- **Never publish related store updates in two `set()` calls** if a surface derives visibility from the first — can unmount an open editor mid-gesture. One gesture = one atomic `set()`.
- **Selector fallbacks must be stable references** — `s.byX[id] ?? []` creates a new array every call and loops Zustand's subscription forever; use a module-level `const EMPTY = []`.
- **A `load()`-style hydration action can lose a race against a mutation for the same key**, even when called after the mutation started (a SELECT-then-INSERT can slot a stale read between the two steps). Fix: a per-key `generation` counter — every mutation bumps it synchronously; `load()` discards its result if the generation moved during its own await. Apply this to any store with more than one independent `load()` caller for the same key.
- **`flexRender(columnDef.cell, ctx)` treats `cell` as a component *type*.** Rebuilding the columns array fresh every render remounts the cell subtree on every re-render, silently resetting local state (e.g. a combobox's `open`). Fix: one stable module-level `cell` function (`renderGridCell`), config + live `nav` passed via TanStack's `meta` option, never captured in a closure.
- **jsdom is missing `hasPointerCapture`/`setPointerCapture`/`releasePointerCapture`, `scrollIntoView`, `window.ResizeObserver`.** Radix/cmdk call these; without polyfills they silently fail to open or throw mid-commit. Polyfilled as no-ops in `src/test/setup.ts`, guarded by environment checks since plain `src/db/*.test.ts` run in `node`, not jsdom.
- **e2e**: two "Undo" buttons exist (app bar + status bar) — scope selectors to `.status-bar`. Dimension names collide with register column headers in plain text locators — scope to `.dim-manager`/`.dim-row__name`. Wait for a phantom row's visible effect before the next keyboard action — the input clears optimistically but the store write may still be in flight.
- **PGlite must stay in `optimizeDeps.exclude`** (vite.config.ts) or its WASM breaks in dev.
- **A bubble-phase `window` keydown listener can be silently swallowed by a descendant's `stopPropagation()`.** `EditableGrid`'s phantom-row input stops propagation unconditionally, so any bubble-phase global shortcut misses right after a commit. Fix: register global shortcuts on the **capture phase**. Only surfaced via e2e — jsdom component tests don't naturally land focus mid-flow the way a browser does.
- **Deferring a global shortcut to native behavior (e.g. text-field undo) must check the field actually has content**, not just that it's focused — an empty focused field has no native undo, so an unconditional guard silently eats the keystroke.
- **A store mutation that cascades into another store's state doesn't need bespoke diffing** — just re-read the affected keys from the DB after the write (safe because it's synchronous-after, not a concurrent `load()`).
- **A `load(id)`-style action must set the id/scope field synchronously, before its own DB round-trip.** `useContextsStore.load(projectId)` only set `projectId` after awaiting its reads; a mutation fired fast enough after mount saw it still `null` and silently no-op'd. **100% reproducible on CI, 0% locally** (CI was consistently slow enough to lose the race). Root-caused via temporary `console.log` + a downloaded Playwright trace, not by guessing from timeouts. Fix: `set({ projectId })` as the literal first line. Apply alongside the generation-counter pattern above, not instead of it.
- **When an e2e failure won't reproduce locally under any configuration, don't keep guessing** — add `trace: 'retain-on-failure'` + `screenshot: 'only-on-failure'` to `playwright.config.ts` and an `upload-artifact` CI step, then `gh run download <id>` and read the trace's JSONL events directly. Found two real CI-only bugs this way after guesswork failed twice.
- **`d3.arc()()` always emits path data centered at `(0,0)`** — no center option, unlike every other geometry helper here. Forgetting to wrap the arc `<path>` in `transform="translate(...)"` renders a barely-visible sliver instead of a circle; unit tests (path non-emptiness only) never caught it, only a real screenshot did.
- **d3-force's collision resolution falls back to `Math.random()` only when two nodes start at the exact same coordinate** — a real risk for a zero-binding context or duplicate-tuple contexts. Fixed by seeding every node's initial position with a deterministic hash of its own id, so exact coincidence never happens and the random path is never exercised (ADR-0005 forbids randomness). Also: the default 300-tick d3-force schedule was too slow for a 100-context/16ms perf budget under CI load — even 60 ticks measured 16.4ms on GitHub Actions vs. ~14ms locally; settled on 30 ticks + a scaled `alphaDecay`, and widened the test's own threshold to 40ms (a strict 16ms doesn't survive shared CI hardware regardless of tick count).
- **A CSS `@container` query cannot target the element its own `container-type` is set on** — only descendants can respond. Silently never matches; no error. Put `container-type` on the parent.
- **Reusing an existing CSS class for a new, differently-shaped layout can silently wreck it** with zero failing tests — jsdom doesn't lay out real pixels. `DesignSurface`'s canvas+register row briefly inherited `.projects`'s `max-width: 560px` this way. Check what a shared wrapper's existing classes were designed for.
- **Manual browser verification is not optional for new visual/layout work**, even with full test coverage — component tests can assert the right *data* but not wrong *geometry* or *CSS layout*. Drive the real dev server with Playwright (`chromium` from the already-installed `playwright-core`, since only `@playwright/test` is a direct dependency) and look at real screenshots.
- **An `opacity: 0` resting state + `animation-fill-mode: forwards` breaks under this repo's blanket `prefers-reduced-motion` rule** (`animation: none !important` in `base.css`) — disabling the animation strips away the only thing that ever sets the visible opacity, permanently stranding the element invisible. Correct shape: resting opacity is the correct *final* state (`1`); the animation only dips it to 0 at mount. Ask "what does this look like with `animation: none` forced" before shipping a mount-fade.
- **A `document`/`window`-level keydown listener meant to defer to a *dynamically-opened* Radix popover has no guaranteed ordering** — same-phase listeners fire in registration order, and a popover's listener registers later (on open) than a shell-level one registered on mount. Don't implement "close popover → clear selection" precedence as two independent global listeners hoping one wins; issue 009 instead wired Escape locally per-node plus a click-canvas-background-to-deselect fallback. For true cross-popover precedence, extend DimensionManager's `onEscapeKeyDown` guard pattern instead.
