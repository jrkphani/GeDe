## Gotchas already paid for (don't rediscover)

- **RTL auto-cleanup doesn't run** without vitest globals â€” `src/test/setup.ts` registers it manually. Component tests need `// @vitest-environment jsdom` as line 1.
- **Radix popovers close on Escape at document level** â€” in-place editors inside popovers need an `onEscapeKeyDown` guard (see DimensionManager) to honor the Esc order.
- **Never publish related store updates in two `set()` calls** if a surface derives visibility from the first â€” can unmount an open editor mid-gesture. One gesture = one atomic `set()`.
- **Selector fallbacks must be stable references** â€” `s.byX[id] ?? []` creates a new array every call and loops Zustand's subscription forever; use a module-level `const EMPTY = []`.
- **A `load()`-style hydration action can lose a race against a mutation for the same key**, even when called after the mutation started (a SELECT-then-INSERT can slot a stale read between the two steps). Fix: a per-key `generation` counter â€” every mutation bumps it synchronously; `load()` discards its result if the generation moved during its own await. Apply this to any store with more than one independent `load()` caller for the same key.
- **`flexRender(columnDef.cell, ctx)` treats `cell` as a component *type*.** Rebuilding the columns array fresh every render remounts the cell subtree on every re-render, silently resetting local state (e.g. a combobox's `open`). Fix: one stable module-level `cell` function (`renderGridCell`), config + live `nav` passed via TanStack's `meta` option, never captured in a closure.
- **jsdom is missing `hasPointerCapture`/`setPointerCapture`/`releasePointerCapture`, `scrollIntoView`, `window.ResizeObserver`.** Radix/cmdk call these; without polyfills they silently fail to open or throw mid-commit. Polyfilled as no-ops in `src/test/setup.ts`, guarded by environment checks since plain `src/db/*.test.ts` run in `node`, not jsdom.
- **e2e**: two "Undo" buttons exist (app bar + status bar) â€” scope selectors to `.status-bar`. Dimension names collide with register column headers in plain text locators â€” scope to `.dim-manager`/`.dim-row__name`. Wait for a phantom row's visible effect before the next keyboard action â€” the input clears optimistically but the store write may still be in flight.
- **PGlite must stay in `optimizeDeps.exclude`** (vite.config.ts) or its WASM breaks in dev.
- **A bubble-phase `window` keydown listener can be silently swallowed by a descendant's `stopPropagation()`.** `EditableGrid`'s phantom-row input stops propagation unconditionally, so any bubble-phase global shortcut misses right after a commit. Fix: register global shortcuts on the **capture phase**. Only surfaced via e2e â€” jsdom component tests don't naturally land focus mid-flow the way a browser does.
- **Deferring a global shortcut to native behavior (e.g. text-field undo) must check the field actually has content**, not just that it's focused â€” an empty focused field has no native undo, so an unconditional guard silently eats the keystroke.
- **A store mutation that cascades into another store's state doesn't need bespoke diffing** â€” just re-read the affected keys from the DB after the write (safe because it's synchronous-after, not a concurrent `load()`).
- **A `load(id)`-style action must set the id/scope field synchronously, before its own DB round-trip.** `useContextsStore.load(projectId)` only set `projectId` after awaiting its reads; a mutation fired fast enough after mount saw it still `null` and silently no-op'd. **100% reproducible on CI, 0% locally** (CI was consistently slow enough to lose the race). Root-caused via temporary `console.log` + a downloaded Playwright trace, not by guessing from timeouts. Fix: `set({ projectId })` as the literal first line. Apply alongside the generation-counter pattern above, not instead of it.
- **When an e2e failure won't reproduce locally under any configuration, don't keep guessing** â€” add `trace: 'retain-on-failure'` + `screenshot: 'only-on-failure'` to `playwright.config.ts` and an `upload-artifact` CI step, then `gh run download <id>` and read the trace's JSONL events directly. - **`d3.arc()()` always emits path data centered at `(0,0)`** â€” no center option, unlike every other geometry helper here. - **d3-force's collision resolution falls back to `Math.random()` only when two nodes start at the exact same coordinate** â€” a real risk for a zero-binding context or duplicate-tuple contexts. - **A canvas-scoped surface that isn't remounted on canvas change keeps stale derived state** (issue 011): `DesignSurface` is keyed by `projectId`, so drilling in/out (same project, different `contextId`) re-renders rather than remounts â€” `guided` and the seed/stale banner state persist. - **An `opacity: 0` resting state + `animation-fill-mode: forwards` breaks under this repo's blanket `prefers-reduced-motion` rule** (`animation: none !important` in `base.css`) â€” disabling the animation strips away the only thing that ever sets the visible opacity, permanently stranding the element invisible.
