# Issues

One markdown file per issue: `NNN-short-slug.md`. Each issue is a **vertical slice** ‚Äî schema ‚Üí store ‚Üí UI ‚Äî sized for TDD: the *Test-first plan* section lists the red tests to write before any implementation, and the acceptance criteria are those tests passing plus the standing gates (`npx tsc --noEmit`, `npx eslint . --quiet`).

## Working agreement (TDD)

1. Pick the lowest-numbered OPEN issue whose blockers are SHIPPED.
2. Write the issue's *Test-first plan* tests; watch them fail.
3. Implement until green; refactor; run `npm run verify`.
4. Update the issue status; commit with the issue number in the message.
